# 프로세스의 이해

## 프로세스란?
**실행 중인 프로그램** (메모리에 올라온 상태)

## 프로그램 vs 프로세스
| 구분 | 프로그램 | 프로세스 |
|------|----------|----------|
| 상태 | 정적 (디스크) | 동적 (메모리 실행 중) |
| 자원 | 없음 | CPU, 메모리 보유 |

## 프로세스 메모리 구조
┌─────────┐ 높은 주소
│  Stack  │ ← 함수 호출, 지역변수
│    ↓    │
│  (빈공간)│
│    ↑    │
│  Heap   │ ← 동적 메모리 할당
├─────────┤
│  Data   │ ← 초기화된 전역변수
├─────────┤
│  Text   │ ← 프로그램 코드
└─────────┘ 낮은 주소

## PCB (Process Control Block)
OS가 프로세스를 관리하기 위한 자료구조
- PID, 상태, PC값, 레지스터, 메모리 정보 포함

> 💡 프로세스는 실행 중인 프로그램이며, OS는 PCB로 각 프로세스를 관리한다.
