# 인터럽트 (Interrupt)

## 인터럽트란?
CPU가 작업 중 **긴급 이벤트 발생 시** 현재 작업을 잠시 멈추고 처리 후 복귀하는 메커니즘

## 종류
### 하드웨어 인터럽트
- 외부 장치 → CPU 신호
- 예: 키보드 입력, 타이머, 디스크 I/O 완료

### 소프트웨어 인터럽트 (트랩)
- 프로그램 실행 중 발생
- 예외: 0으로 나누기, 메모리 접근 오류
- 시스템 콜: OS 서비스 요청

## 처리 과정

인터럽트 발생
현재 CPU 상태 저장 (레지스터, PC → 스택)
인터럽트 벡터 테이블에서 ISR 주소 조회
ISR(Interrupt Service Routine) 실행
저장된 상태 복원
중단된 작업 재개


## 인터럽트 vs 폴링
| 방식 | 특징 |
|------|------|
| 인터럽트 | 이벤트 발생 시 신호 → CPU 효율↑ |
| 폴링 | 주기적 상태 확인 → CPU 낭비 |

> 💡 인터럽트는 CPU가 비동기 이벤트를 효율적으로 처리하는 핵심 메커니즘이다.
